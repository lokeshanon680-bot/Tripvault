# TripVault

> Full-stack travel journaling app with user auth, trip management, and photo uploads (React + Vite + Express + MongoDB).

## Features

- User registration & login (JWT)
- Create, read, update, delete trips
- Photo uploads via Cloudinary
- Public user profiles

## Prerequisites

- Node.js 16+ and npm
- MongoDB (local or Atlas)
- Cloudinary account (for photo uploads)

## Project structure

- `src/` — React (Vite) frontend
- `server/` — Express API, controllers, routes, models, middleware
- `api/` — serverless API entry (used for Vercel deployment)
- `dist/` — production build output (generated by `npm run build`, not edited manually)

## Install

1. Clone the repo
2. Install root dependencies:

```
npm install
```

3. Install server dependencies:

```
cd server && npm install
```

## Environment

Create a `.env` file inside `server/` with:

```
PORT=5000
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_jwt_secret
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
```

## Run (development)

Start the server and client in separate terminals:

```
# Start server
cd server && npm start

# Start frontend (Vite)
npm run dev
```

## Build & Deploy

```
npm run build
```

Deployed via Vercel — see `vercel.json` for routing/build config.
Live demo: https://tripvault-blue.vercel.app

## API (quick reference)

- `POST /api/auth/register` — register a user
- `POST /api/auth/login` — obtain JWT
- `GET  /api/trips` — list trips (auth protected)
- `POST /api/trips` — create trip (auth protected)
- `PUT  /api/trips/:id` — update trip (auth protected)
- `DELETE /api/trips/:id` — delete trip (auth protected)
- `POST /api/upload` — upload a photo to Cloudinary (auth protected)

## Notes

- For production, configure a real MongoDB URI and a secure `JWT_SECRET` — there is no JSON-file fallback mode.
- `dist/` is a build artifact; regenerate it with `npm run build` rather than editing directly.
